# 国盛睿股票代码跳转功能

## 目标

在桌面股票助手里点击股票代码后，自动唤出本机已安装的国盛睿，并在国盛睿里输入该股票代码，让国盛睿跳转到对应股票界面。

该功能只做看盘和人工交易窗口联动，不做自动下单、撤单、交易 API 调用，也不读取或保存国盛睿账号、密码。用户点击“扫描国盛睿持仓”时，软件会导出并读取国盛睿资金股份查询文本，用于同步本地持仓和预算系统。

## 背景判断

当前未发现国盛睿提供给普通外部软件使用的公开 API 或 URL Scheme。国盛睿本质上是通达信系 PC 终端，本机安装信息如下：

- 软件：国盛睿 V1.49
- 主程序：`C:\zd_gszq_gm\TdxW.exe`
- 窗口类名线索：`TdxW_MainFrame_Class`

因此采用 Windows 本地自动化方案：聚焦国盛睿窗口，然后模拟输入股票代码并按回车。

## 用户行为

1. 用户在软件表格里点击股票代码。
2. 软件复制股票代码到剪贴板，作为兜底。
3. 软件查找正在运行的国盛睿窗口。
4. 如果国盛睿已运行，切到前台。
5. 如果国盛睿未运行，启动 `TdxW.exe`，等待主窗口出现后切到前台。
6. 软件向国盛睿输入 6 位股票代码，并发送回车。
7. 国盛睿通过键盘精灵跳转到对应股票界面。

## 完成标准

- 点击 `600000` 这类沪市代码后，国盛睿前台显示对应股票。
- 点击 `000001`、`300xxx`、`688xxx` 等代码也能跳转。
- 国盛睿未打开时，能自动启动并跳转。
- 国盛睿启动失败或窗口未找到时，软件给出明确提示，并保留“已复制代码”的兜底结果。
- 不弹出交易委托界面，不触碰 F12、买入、卖出等交易相关入口。

## 实现方案

新增一个本地适配函数，例如：

```python
def open_guoshengrui_for_ticker(ticker_text: str) -> bool:
    """Focus Guoshengrui and input a stock code. Returns True on best-effort success."""
```

核心流程：

1. 标准化代码

```python
ticker = "".join(ch for ch in ticker_text if ch.isdigit())[:6]
if len(ticker) != 6:
    return False
```

2. 定位主程序

优先使用固定路径：

```text
C:\zd_gszq_gm\TdxW.exe
```

后续可增加注册表检测：

```text
HKCU/HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall
```

3. 查找或启动窗口

- 优先查找进程名：`TdxW.exe`
- 优先匹配窗口标题包含：`国盛睿`
- 可选匹配窗口类名：`TdxW_MainFrame_Class`
- 未找到时启动 `TdxW.exe`
- 启动后轮询等待窗口，建议超时 10-15 秒

4. 前置窗口并输入

0.4.44 起不再无条件 `SW_RESTORE`，只在国盛睿最小化时还原，避免点击代码把最大化窗口缩成普通窗口。

```python
if win32gui.IsIconic(hwnd):
    win32gui.ShowWindow(hwnd, win32con.SW_RESTORE)
else:
    win32gui.ShowWindow(hwnd, win32con.SW_SHOW)
win32gui.SetForegroundWindow(hwnd)
```

输入方式也改为代码前缀自适应：

- `600/688/9xx` 优先使用数字小键盘事件，避免普通数字 `60` 被国盛睿解释成周期/功能快捷键。
- `300xxx` 优先使用主键盘数字事件，避免小键盘 `30` 进入行情报价列表。
- 其他 6 位代码先用主键盘，疑似跳到“全部Ａ股”等报价页时自动换另一种输入方式重试。
- 剪贴板仍写入 6 位代码作为兜底，但不再依赖 `Ctrl+V`，因为图表窗格可能吞掉粘贴事件。

  - 目标行为：点击“代码”列复制并跳转国盛睿

## 交易弹窗联动

弹窗里的买入/卖出按钮使用同一个本地自动化桥接，但边界更严格。本轮修正起，交易按钮不再复用底部交易面板作为成功条件，而是强制走分时图右键触发：

1. 先跳转到对应股票图表，并把 6 位代码写入剪贴板。
2. 聚焦目标股票主图区域。实测国盛睿 V1.49 在已处于分时图时点击顶部 `分时` 标签会切回报价表，因此自动化不再机械点击该标签。
3. 在目标股票主图区域右键，选择图表菜单里的：
   - `闪电买入(1)`
   - `闪电卖出(2)`
4. 只有检测到 `闪电买入` 或 `闪电卖出` 独立窗口，并且窗口内能看到目标 6 位代码时才返回成功；若弹窗不是目标代码，会立即判失败。

该流程不向交易弹窗输入证券代码，不输入价格、不按回车、不点击 `买入` / `卖出` 提交按钮。买入弹窗可在用户输入或扫描账户可用现金、当前持仓市值和总资产后自动填入买入数量，数量计算规则为：

- 总资产优先使用国盛睿资金股份查询导出的 `资产` 字段；如果没有扫描到该字段，则回退为 `可用现金 + 当前持仓市值`。
- 目标仓位金额 = 总资产 × 扫描结果的建议资金占比。
- 已有同代码持仓会先占用目标仓位金额。
- 可用现金作为账户级买入上限。
- 国盛睿闪电窗口里的 `最大可买` 作为现金上限。
- 最终数量按 100 股一手向下取整。

若已有闪电下单窗口，则不会覆盖该窗口，用户需要先处理或关闭后再重新点击。

## 持仓同步

持仓管理页新增 `扫描国盛睿持仓`：

1. 聚焦国盛睿。
2. 打开交易/持仓面板并切到 `持仓`。
3. 点击 `输出`，使用国盛睿默认生成的 `资金股份查询.txt`。
4. 解析：
   - `可用` -> 预算系统的可用现金。
   - `参考市值` -> 预算系统的当前持仓市值。
   - `资产` -> 预算系统的账户总资产。
   - 持仓表 -> 软件本地持仓。
5. 本地持仓采用更新/新增策略，不自动删除导出结果中未出现的本地持仓。

新导入持仓若没有既有管理线，默认以成本价生成参考管理参数：目标上沿 `成本 * 1.10`，硬止损 `成本 * 0.96`，移动止盈回撤 `3%`。

若券商同步进来的持仓已经明显盈利，系统不再沿用过期的 `成本 * 1.10`，也不使用 `当前高点 * 1.10` 外推目标。目标上沿按持仓成本进入利润阶梯，保护利润则按近期价格/持仓高点动态上移：

- 当前盈利 `< 20%`：按 10% 成本阶梯推进。
- 当前盈利 `20%-60%`：按 15% 成本阶梯推进。
- 当前盈利 `60%-120%`：按 20% 成本阶梯推进。
- 当前盈利 `> 120%`：按 25% 成本阶梯推进。

动态保护线取 `成本阶梯保护线`、`近期高点回撤线` 和已有有效保护线中的较强者。已经进入管理状态的强势持仓到达目标上沿时，不再直接全仓止盈，而是滚动上移下一档目标和保护线；只有跌破动态保护线时才提示保护利润/减仓。

例如成本 `2.3448`、当前/最高 `8.38` 的持仓，盈利约 `257%`，下一目标阶梯为成本 `+275%`，目标上沿约 `8.7930`；保护线取成本 `+250%` 与近期高点回撤线中的较强者，约 `8.2068`。

## 风险与边界

- 这是本地窗口自动化，不是官方公开 API；国盛睿升级后窗口标题或输入行为可能变化。
- 如果前台窗口切换被 Windows 权限限制，可能需要用户手动点一下国盛睿窗口。
- 如果国盛睿正在弹登录框、更新框、风险提示框，输入代码可能无法落到键盘精灵。
- 不应在交易委托窗口中自动输入证券代码、价格或确认键；只允许在买入弹窗中填入计算后的数量，避免误填交易字段。
- 持仓同步依赖国盛睿 `资金股份查询` 的导出文本；国盛睿升级后如果导出格式变化，解析可能需要调整。

## 验证清单

- 国盛睿已运行，点击代码能跳转。
- 国盛睿未运行，点击代码能启动并跳转。
- 国盛睿存在弹窗时，不误输入交易字段，失败时给提示。
- 无效代码不会触发跳转。
- 点击名称仍打开雪球页面。
- 持仓同步能读取可用现金、参考市值、资产和持仓明细，并写入软件持仓。
